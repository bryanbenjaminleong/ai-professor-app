# AI Professor — Content Strategy & Master Template

**Version:** 2.0 — March 31, 2026
**Philosophy:** Quality over quantity. Every piece of content must be so good that people buy the next one because they *want* to, not because they're subscribed.

---

## Revenue Model: One-Time Purchase (No Subscriptions)

| Tier | Type | Price | What You Get |
|------|------|-------|-------------|
| **Level 1** | Quick Guides | **FREE** | 3–5 lessons, ~30 min, get productive fast |
| **Level 2** | Full Courses | **$14.99** | 8–12 lessons, projects, assessments, certificate |
| **Level 3** | Programs | **$49.99** | Full program (all modules + capstone) |

**Why one-time purchase:**
- Users buy what they need, when they need it
- Stickiness comes from content quality, not lock-in
- Each purchase is a quality statement — they'll come back for more
- No pressure to produce filler content to "justify" a subscription

### How Courses & Programs Overlap

**Every program module is also a standalone course.** Same content, two storefronts.

| Where They See It | What They Pay | What They Get |
|-------------------|---------------|---------------|
| Browsing **Courses** | $14.99 | Single module (e.g. "Risk Management") |
| Browsing **Programs** | $49.99 | All modules + capstone project (save $25) |

**Example — Digital Transformation Program:**

| # | Module (also sold as standalone course) | Standalone | In Program |
|---|----------------------------------------|------------|------------|
| 1 | AI Strategy for Leaders | $14.99 | ✅ Included |
| 2 | Risk Management & AI Governance | $14.99 | ✅ Included |
| 3 | Data-Driven Decision Making | $14.99 | ✅ Included |
| 4 | Building AI Workflows | $14.99 | ✅ Included |
| 5 | Capstone: Digital Transformation Plan | — | ✅ Included |
| | **Total if bought separately** | **$59.96** | **$49.99** |

**This means:**
- Create a module once → it appears in both Courses and Programs
- More SKUs without more content work
- Lower barrier to entry ($14.99) with upsell to program ($49.99)
- Users who bought a standalone module can upgrade to the full program for the difference

### Cross-Linking Rules (MANDATORY)

**Course description (standalone module) MUST include:**
> 📦 This course is also **Module [N]** in the **[Program Name]** program ($49.99 for all modules + capstone). [View Program →]

**Program description MUST include:**
> Each module in this program is also available as a standalone course ($14.99 each):
> - Module 1: [Course Title] — $14.99 standalone or included in program
> - Module 2: [Course Title] — $14.99 standalone or included in program
> - ...
> 
> 💰 **Save $[X]** when you enroll in the full program instead of buying modules separately.

**Module quality standard:** Every module must be complete and valuable on its own — someone who buys just that one module must feel they got their money's worth, not that they're getting a "preview" of a program.

## Master Content Template

### Universal Principles (applies to ALL content)

Every lesson, guide, and course must include:

1. **Clear goal** — What will the student accomplish?
2. **Real-world context** — Why does this matter? (not just "you'll learn X")
3. **Step-by-step instructions** — Test them. They must work.
4. **Working code examples** — Not pseudocode. Real, runnable code.
5. **Common mistakes** — What goes wrong and how to fix it
6. **Try it yourself** — At least 1 hands-on exercise per lesson
7. **Key takeaways** — 3–5 bullet summary
8. **Next steps** — Where to go from here (linked to related content)

## Quality Checklist (before ANY content is published)

### Content Quality
- [ ] Every lesson meets minimum character count (guides: 1,500+ / courses: 5,000+)
- [ ] No filler or padding — every sentence adds value
- [ ] Technical accuracy verified (code examples actually work)
- [ ] Real-world scenarios, not abstract theory
- [ ] Progressive difficulty — each lesson builds on the previous
- [ ] No unexplained jargon — define terms on first use

### Structure
- [ ] Learning objectives are specific and measurable
- [ ] Hands-on exercise in every lesson
- [ ] Common mistakes section included
- [ ] Key takeaways at the end
- [ ] "Next steps" linking to related content
- [ ] Consistent formatting across all lessons

### Accuracy
- [ ] All tool names, URLs, and pricing are current
- [ ] Code examples tested and working
- [ ] No outdated information (AI tools change fast — review quarterly)
- [ ] Screenshots/captions match current UI where applicable

### Professional Standards
- [ ] Mobile-friendly formatting (short paragraphs, clear headers)
- [ ] Accessible language (clear, not dumbed down)
- [ ] No grammatical errors
- [ ] Consistent tone: expert but approachable
- [ ] Inclusive language

## Content Creation Workflow

1. **Outline** — Define learning objectives, lesson titles, projects
2. **Draft** — Write full content following the template
3. **Review** — Check against quality checklist
4. **Test** — Verify all code examples work, all links are live
5. **Publish** — Set `is_published: true` only when ALL lessons are complete
6. **Review cycle** — Quarterly audit for accuracy (tools/pricing change fast)
