### Key Lessons:
- Fine-grained PATs can ONLY access repos owned by the token account
- Classic PATs with `repo` scope work for cross-account collaborator access
- Login uses Supabase auth stored in localStorage — NOT NextAuth cookies
- `useSession()` from next-auth is useless when login uses a different auth system
- Vercel CDN caching requires hard refresh or incognito to see new deployments
- The `write` tool can inject garbage content — always verify files after writing
- Always check middleware.ts for hardcoded domain whitelists when changing URLs
- GitHub secret scanning catches tokens in markdown files
